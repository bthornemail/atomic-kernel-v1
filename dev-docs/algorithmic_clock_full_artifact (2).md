# Abstract

This artifact defines a deterministic algorithmic clock and its shared basis extension model.

The clock does not measure physical duration. It defines a reproducible logical position by iterating a fixed transition law over a bounded state space. Given the same basis artifact, canonical seed derivation procedure, width, and step count, any conforming implementation must produce the same state sequence and the same projected witness sequence.

The system is organized in five layers:

1. **Shared basis artifact** — a public, versioned symbolic substrate
2. **Canonical seed derivation** — deterministic reduction from basis + context into a bounded state
3. **Replay law** — a width-parameterized affine circulant state transition
4. **Projection law** — canonical observable features derived from state
5. **Witness / extension layer** — application-specific artifacts derived without changing the lower law

The basis artifact proposed here is the Unicode codepoint space with a root UTF-EBCDIC specification artifact. Unicode supplies a public finite symbolic domain. UTF-EBCDIC supplies an operable reference realization. Together they provide a shared substrate from which canonical seeds may be derived and independently falsified or reproduced by any participant.

# 2. Core Mathematical Object

For each width \(n\), define the clock family member

\[
\mathcal{C}_n = (S_n, \delta_n, \pi_n)
\]

where:

- \(S_n = \{0,1,\dots,2^n-1\}\) is the bounded state space
- \(\delta_n : S_n \to S_n\) is the transition law
- \(\pi_n : S_n \to B_n\) is the canonical projection law

The clock sequence is generated by replaying \(\delta_n\) from a seed \(x_0\):

\[
x_{k+1} = \delta_n(x_k)
\]

The projected witness sequence is:

\[
\pi_n(x_0), \pi_n(x_1), \pi_n(x_2), \dots
\]

---

# 3. State Space

For width \(n\):

\[
S_n = \{0,1,\dots,2^n-1\}
\]

Mask operator:

\[
\operatorname{mask}_n(x) = x \bmod 2^n
\]

This ensures all values remain in the bounded state space.

---

# 4. Rotation Operators

For \(x \in S_n\) and shift \(k\):

Left rotation:

\[
\operatorname{rotl}_n(x,k) = \operatorname{mask}_n\big((x \ll k) \lor (x \gg (n-k))\big)
\]

Right rotation:

\[
\operatorname{rotr}_n(x,k) = \operatorname{mask}_n\big((x \gg k) \lor (x \ll (n-k))\big)
\]

---

# 5. Transition Law

The width-parameterized transition law is:

\[
\delta_n(x) = \operatorname{mask}_n\big(
\operatorname{rotl}_n(x,1)
\oplus
\operatorname{rotl}_n(x,3)
\oplus
\operatorname{rotr}_n(x,2)
\oplus
C_n
\big)
\]

where \(\oplus\) is bitwise XOR and \(C_n\) is a width-matched constant.

## 5.1 Constant Family

The constant is defined by repeating the byte pattern `0x1D` across the chosen width.

Examples:

- \(C_{16} = 0x1D1D\)
- \(C_{32} = 0x1D1D1D1D\)
- \(C_{64} = 0x1D1D1D1D1D1D1D1D\)
- \(C_{128}\) = repeated `0x1D` for 16 bytes
- \(C_{256}\) = repeated `0x1D` for 32 bytes

---

# 6. Affine Circulant Form

Define the linear component:

\[
L_n(x) = \operatorname{rotl}_n(x,1) \oplus \operatorname{rotl}_n(x,3) \oplus \operatorname{rotr}_n(x,2)
\]

Then:

\[
\delta_n(x) = L_n(x) \oplus C_n
\]

This places the clock in the class of affine dynamical systems over \(GF(2)^n\):

\[
\delta_n(x) = A_n x \oplus b_n
\]

where:

- \(A_n\) is the circulant linear operator induced by taps \(\{1,3,n-2\}\)
- \(b_n = C_n\)

Thus the clock is a **width-parameterized affine circulant dynamical system**.

---

# 7. Replay Law

Given a seed \(x_0\), replay is defined by iteration:

\[
x_{k+1} = \delta_n(x_k)
\]

The replay sequence is:

\[
O_n(x_0) = (x_0, x_1, x_2, \dots)
\]

with:

\[
x_k = \delta_n^k(x_0)
\]

This sequence is deterministic and reproducible by any conforming implementation.

---

# 8. Projection Law

The canonical projection maps a state to a structural witness tuple called a **band**.

\[
\pi_n(x) = (w(x), d(x), e(x))
\]

where:

## 8.1 Width

\[
w(x) =
\begin{cases}
0 & x = 0 \\
\lfloor \log_2(x) \rfloor + 1 & x > 0
\end{cases}
\]

## 8.2 Density

\[
d(x) = \text{popcount}(x)
\]

## 8.3 Texture

\[
e(x) = \sum_{i=0}^{n-1} \mathbf{1}(\text{bit}_i(x) \ne \text{bit}_{(i+1)\bmod n}(x))
\]

The witness sequence is therefore:

\[
T_n(x_0) = (\pi_n(x_0), \pi_n(x_1), \pi_n(x_2), \dots)
\]

---

# 9. Orbit Signature

For a seed \(x\), define the canonical orbit signature:

\[
\Omega_n(x) = (\lambda(x), D(x), E(x))
\]

where:

- \(\lambda(x)\) = cycle length of the orbit containing \(x\)
- \(D(x)\) = density distribution along the orbit
- \(E(x)\) = band transition structure along the orbit

This signature is a deterministic invariant of the replay process for fixed width, seed law, and basis artifact.

# 10. Order Theorem for Power-of-Two Widths

Let \(n = 2^k\) with \(k \ge 4\), and let \(A_n\) be the circulant linear operator induced by the tap set \(\{1,3,n-2\}\).

Then the operator order satisfies:

\[
\operatorname{ord}(A_n) = \frac{n}{2}
\]

for the tested widths:

- \(n=16 \Rightarrow \operatorname{ord}(A_{16}) = 8\)
- \(n=32 \Rightarrow \operatorname{ord}(A_{32}) = 16\)
- \(n=64 \Rightarrow \operatorname{ord}(A_{64}) = 32\)
- \(n=128 \Rightarrow \operatorname{ord}(A_{128}) = 64\)
- \(n=256 \Rightarrow \operatorname{ord}(A_{256}) = 128\)

This yields two comparison ladders:

## 10.1 Power Ladder

\[
16 \to 32 \to 64 \to 128 \to 256
\]

with operator orders:

\[
8 \to 16 \to 32 \to 64 \to 128
\]

## 10.2 Jump Ladder

\[
16 \to 64 \to 256
\]

with operator orders:

\[
8 \to 32 \to 128
\]

These ladders provide a deterministic width hierarchy for comparing orbit depth, replay structure, density distributions, band transitions, and other derived observables.

# 11. Meaning of the Clock

This artifact does not define physical time. It defines a reproducible logical coordinate system.

For any event associated with a replay step \(k\), the step difference between events is a logical distance:

\[
\Delta(A,B) = k_B - k_A
\]

Thus the clock provides a deterministic coordinate system over state evolution. It allows participants to compare the relative positions of events by counting iterations of the transition law rather than by consulting physical clocks.

The purpose of the basis artifact is not to redefine Unicode. Its purpose is to supply a stable public symbolic substrate from which deterministic seeds can be derived.

Unicode is chosen because it gives:

- a globally shared symbol space

UTF-EBCDIC is chosen as a root artifact because it gives:

- a concrete encoding realization

Together they provide:

\[
\text{shared symbolic domain} + \text{shared operable artifact}
\]

This is sufficient to support canonical seed derivation in a decentralized environment.

# 14. Canonical Seed Derivation

Define a canonical seed derivation map:

\[
\Sigma_n : B \times \mathcal{X} \to S_n
\]

where:

- \(B\) is the shared basis artifact
- \(\mathcal{X}\) is an application-defined context space
- \(S_n\) is the chosen bounded state space

The derivation pipeline is:

\[
(B, x) \to \text{canonicalize}(B, x) \to \text{encode}(B, x) \to \text{reduce}_n(B, x)
\]

A canonical form is obtained from basis + context. That form is encoded through the agreed basis artifact and reduced into width \(n\).

A generic form is:

\[
\Sigma_n(B,x) = \operatorname{mask}_n(H(\operatorname{canon}(B,x)))
\]

where:

- \(\operatorname{canon}\) is a canonicalization procedure
- \(H\) is a deterministic reduction or digest function
- \(\operatorname{mask}_n\) maps into the width-bounded state space

The crucial property is not the specific digest, but that the entire derivation process is normative and reproducible.

# 15. Unicode Embedding View

For each width \(n\), states live in \(GF(2)^n\). Unicode codepoints live in the integer domain \(U\).

An embedding:

\[
\phi_n : GF(2)^n \to U

may be used to interpret bounded states or projected witnesses as codepoint-valued artifacts when the chosen width and range allow.

This does not make Unicode the clock. Rather:

- the clock is the replay law
